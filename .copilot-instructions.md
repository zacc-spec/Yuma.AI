# Yuma.AI - Claude Copilot Instructions

## Project Overview
Yuma.AI is a full-stack Python application with frontend/backend separation. This file guides Claude Copilot on how to work effectively with this codebase.

## Project Structure
- **backend/**: Flask Python application with Claude AI integration
- **frontend/**: React-based frontend application
- **packages/dependencies/**: All external Python dependencies (requirements.txt)

## Key Technologies
- **Backend**: Python 3.10+, Flask, SQLAlchemy, Anthropic Claude API
- **Frontend**: React, Vite, Axios
- **Infrastructure**: Docker, Kubernetes (Minikube), Helm (optional)

## Development Workflow

### Python Backend
1. Always use `packages/dependencies/requirements.txt` for dependency specifications
2. Follow PEP 8 style guide (use black formatter)
3. Keep Flask routes modular and well-documented
4. Use environment variables from `.env` files

### Frontend
1. Use React with functional components and hooks
2. Leverage Vite for fast development and building
3. Keep components focused and reusable
4. Use Axios for API communication

## File Conventions
- Python files: `snake_case.py`
- React components: `PascalCase.jsx` or `PascalCase.tsx`
- Constants: `UPPER_CASE` in dedicated files

## When Adding Features
1. Create feature branch from main
2. Update requirements.txt if adding Python dependencies
3. Update package.json if adding frontend dependencies
4. Document in-code with docstrings (Python) and JSDoc (JavaScript)
5. Ensure both backend and frontend tests pass

## Deployment
- Backend containerized with Docker (Dockerfile in backend/ if exists)
- Frontend built with `npm run build` → static files
- Deployment via Kubernetes manifests in `k8s/` folder (if it exists)

## Common Tasks

### Install Python Dependencies
```bash
pip install -r packages/dependencies/requirements.txt
```

### Run Backend (Debug Mode)
```bash
python backend/main.py
```

### Run Frontend
```bash
cd frontend && npm install && npm start
```

### Format Code
```bash
# Python
black backend/

# JavaScript
prettier --write frontend/
```

## Helpful Context for Claude
- This is an AI-enabled application, so Claude API integration is important
- Think cloud-native: consider Kubernetes deployment scenarios
- Security: Handle API keys securely via environment variables
